# 1. Product Goal

Build **InSureGig**, an AI-inspired parametric insurance platform for gig delivery workers.

The platform should protect delivery workers from **loss of income caused by external disruptions**, such as:

* heavy rain
* severe pollution
* extreme heat
* curfew / local zone closure

Important constraints:

* The platform must cover **income loss only**
* Do **not** include health insurance, life insurance, accident insurance, or vehicle repair
* Pricing must be based on a **weekly premium**
* Claims should be **automatically triggered**
* Payout can be simulated, but the full workflow must be functional

# 11. Dynamic Premium Logic

Use rule-based logic that behaves like AI-inspired pricing.

Example:

* Basic plan base premium = ₹20
* Standard plan base premium = ₹35
* Premium plan base premium = ₹63

Adjustments:

* Heavy Rain Alert = +₹10
* Poor AQI = +₹5
* Safe Zone Discount = -₹2

Examples:

* Mumbai worker + rain risk = higher premium
* Delhi worker + pollution risk = higher premium
* safe zone worker = lower premium

This calculation must be done in backend and sent to frontend.

# 12. Auto-Claim Workflow

Implement a **zero-touch claims flow**.

## Workflow

1. Admin creates disruption event
2. Event becomes active
3. Backend finds workers in affected city
4. Backend checks active policy
5. Backend checks fraud rules
6. Valid claims are auto-created
7. Claim status becomes approved
8. Payout is simulated
9. Worker sees weather alert screen
10. Worker sees payout success screen
